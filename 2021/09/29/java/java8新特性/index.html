java8新特性

一、lambda(参数式函数) 代码更少

fun1(fun2())

  1. 参数式函数，参数式lambda。

  2. java中 lambda是实现了函数的对象，而不像其他语言一样是一个函数。一般用于接口，这个接口只有一个函数需要重写。这个接口的实例就是lambda。

  3. 但是用到的lambda还是写的那个重写的函数，也就是说我们需要某个功能的时候，我们可以根据四大函数式接口写出自己的函数，将这个接口传入，就完成了一个功能器。

  4. 为了完成多功能，所以lambda对象应该是以参数的身份传入到指定函数，以为创建Lambda很简单，代码很少，这样在不改变原函数的情况下就能根据传入的lambda不同而拥有不同的功能，目的就是多态。

     public List filter(int []nums,Predicate<Integer>predicate){
filter(nums,o->o>6);
filter(nums,o->2*o>6);
filter(nums,o->o*o>6);
filter(nums,o->1>6);

  5. 创建比较器

    //lambda
        Comparator<Integer>comparator2=(o1,o2)->o1.compareTo(o2);
        //方法引用  lambda 传fun
        Comparator<Integer>comparator3= Integer::compareTo;
Arrays.sort(nums,comparator3);

  6. 创建进程

    Runnable runnable2=()-> System.out.println("rrrr2");

  7. lambda优化

    1. 去掉形参类型(类型推断)

      1. 直接传入o1,而不用在前面加integer

      2. int[]nums=new int[]{1,2,3}–>

        int[]nums={1,2,3}

    2. 去掉return和{} 只有一句的情况

    3. 只有一个参数，可以去掉小括号

  8. 格式

    1. 一个参数没返回:obj=x->fun(x)
    2. 无参无返回: obj=()->fun()
    3. 两个参数: obj=(x,y)->fun(x,y)
    4. 有返回: obj=(x,y)->{fun(x,y) return}

二、函数式接口

  1. @FuntionalInterface

  2. 面向函数范式编程(FOP)(SQL)

  3. 面向接口编程

  4. 面向过程编程（Procedure Oriented、简称PO） 和 面向对象编程（Object Oriented、简称OO） 我们一定听过，然而实际企业级开发里受用更多的一种编程思想那就是：面向接口编程（Interface-Oriented）！

  5. 四大函数式接口

    1. Consumer

      ​ 消费型：有参无返回

      ​ void accept(T t)

    2. Suppiier

      ​ 供给型：无参有返回

      ​ T get()

    3. Functional<T,R>

      ​ 函数型：处理参数

      ​ R apply(T t)

    4. Prediate

      ​ 断定型：有参判定返回bool

      ​ boolean test(T t)

三、方法引用 构造器引用

  1. 当要传递给lambda操作，已经有实现方法，就可以使用方法引用。

  2. 基于lambda

  3. 关键：传入的方法的返回类型和参数相同

  4. 就好像现在不用自己写函数传入Lambda体了，而是有现成的函数，那么直接传入

    类::方法名就ok了。

  5. 三种情况+一种构造器

    1. 类::非静态方法
    2. 对象::静态方法
    3. 类::非静态方法
    4. 构造器引用
      1. 类::new

  6. 要求：两个函数除了名字不一样，其他都一样

  7. 细节

    1. 不用写入形参，因为类型与参数推断，且要求两函数是一样的，所以没必要写形参。

四、stream api 并行计算

  1. stream:运算 cpu打交道

    Collection:集合 内存打交道

  2. 类似于SQL语句

  3. 作用：

    1. 查询数据不能用sql的情况,如nosql需要把查询到的数据在java里面进行处理

  4. 特点：

    1. 自己不储存(视图)
    2. 不会改变原对象，也就是inplace=False
    3. 操作延迟，直到终止操作才会执行

  5. 步骤

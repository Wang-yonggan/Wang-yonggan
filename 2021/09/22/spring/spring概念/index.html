一、基本概念

Spring是

1. 核心

  1. IOC 控制反转

    1. DI 依赖注入

  2. AOP 面向切面

    1. proxy 代理

2. 优点

  1. 降低组件之间的耦合度（互相依赖）

  2. 方便开发，降低复杂度: 提供了很多已经实现的辅佐组件

  3. 提供了AOP技术，利用它很容易实现如权限拦截，运行期监控等功能 

  4. IOC技术容器提供单例模式支持，开发人员不需要自己编写实现代码.

  5. 方便整合其他框架

  6. spring源码是

  7. 方便事务操作

二、IOC(控制反转)

1. IOC介绍

  1. 平常

  2. 现在：引入Ioc，将创建对象的控制权交给spring的ioc，如果需要使用，就用DI(依赖注入)@Autowired.

  1. 优点：集中管理对象，方便维护，降低了耦合度。

  2. IOC和DI区别

    1. ioc控制了对象创建的权力。

    2. di是ioc的实现的一步骤。

  3. 哪些类放入ioc

    1. 放入：工具类，dao,service,controller
    2. 不放人：实体类，servlet,listener,filter

2. IOC配置Bean

  1. xml配置：<bean class:path>

    1. 属性

      1. set +<properity name value>
      2. set+<properity name ref>
      3. 构造注入：<constructor-arg>+index

    2. 代码

  2. 注解@Component(@Controller,@Service,@Repostory)

  3. javaConfig:@Configtursion+@Bean，这像是自己实现的ioc，更灵活。

  4. @import

3. IOC获取Bean

ioc两种获取(接口)

  • BeanFactory 内置

    • 懒加载，加载配置文件不会创建对象，只有getBean才会创建bean，

  • ApplicationContext

    • 饿汉式，加载配置文件的时候就会创建对象
    • 实现类

4. ioc原理

  1. 简单工厂+反射

  2. code:beanfactory.getBean+(强制类型转换+Class.forName)

  3. ioc实现原理DI技术–>map.put(name,new)

  4. ioc的对象是创建ioc的时候一起同时创建的，不是启动时创建的，也不是使用的时候创建的。

5.紧耦合 松耦合(DIP)

  1. 紧耦合：类之间的高度依赖
  2. 松耦合：单一职责原则–>接口分离原则–>依赖倒置原则。

二、Bean

1. bean概念

2.1 IOC bean 行为

  • 一类是注册Bean

    ​ @Component , @Repository , @ Controller , @Service , @Configration这些注解都是把你要实例化的对象转化成一个Bean，放在IoC容器中，等你要用的时候，它会和上面的@Autowired , @Resource配合到一起，把对象、属性、方法完美组装。

  • 一类是使用Bean

    即是把已经在xml文件中配置好的Bean,或者手动注册的Bean拿来用，完成属性、方法的组装；比如@Autowired , @Resource，可以通过byTYPE（@Autowired）、byNAME（@Resource）的方式获取Bean；

2.2 Bean属性
  1. bean元素:使用该元素描述需要spring容器管理对象
  2. name属性:给被管理的对象起个名字,获得对象时getBean(“name值”)
  3. Class/Type属性:被管理对象的完整类名
  4. id属性:与name属性一模一样，名称不可重复，不能使用特殊字符

3.1 scope属性

​ 绝大多数情况下，使用单例singleton(默认值)，但是在与struts整合时候，务必要用prototype多例，因为struts2在每次请求都会创建一个新的Action，若为单例，在多请求情况下，每个请求找找spring拿的都是同一个action。

  1. singleton 默认值
    单例对象 :被标识为单例的对象在spring容器中只会存在一个实例
  2. prototype
    多例原型:被标识为多例的对象,每次在获得才会被创建,每次创建都是新的对象
  3. request
    Web环境下,对象与request生命周期一致
  4. session
    Web环境下,对象与session生命周期一致

三、AOP

​ aop就是将动态代理规范化，约定大于配置。

1.动态代理

  1. 作用：创建对象，不改变类的情况下实现功能增强。

  2. 两种实现方式：

    1. JDK
      1. 要求目标类有接口
      2. 若没有接口。可用CGlib，
    2. Proxy

2. AOP四要素

  1. aspect:切面，增强的功能的代码。

    三要素：

    1. 功能代码函数def doLog()
    2. 代码位置(切入点)，一般就是业务代码函数。execution(path.def)
    3. 执行时间（before/after）

  2. joinpoint:连接点，业务代码。

  3. pointCut切入点,连接点的结合

  4. advice 通知，表示切面功能执行的时间

3. AOP实现

  1. spring,事务处理常用aop（代码多，不常用）
  2. aspectJ:专业aop框架。spring集成了这个框架。
    1. xml实现：用于事务
    2. 注解实现：常见环境下都用注解。
      1. @Before
      2. @AfterReturn
      3. @Around
      4. @AfterThrowing
      5. @After
    3. 切入点表达式execution(public pString]* [com.wyg.user.]getName(..)* exception)

4. AOP实现步骤:

  1. 加入依赖spring,aspect

  2. 创建目标类(接口，实现)

  3. 创建切面类

    1. 加如@aspect
    2. 定义增强功能方法，并在方法上加入注解

  4. 创建spring配置文件：声明对象，交予ioc管理 加入aop标志生成代理类aop:aspectj-autoproxy/。

  5. xml配最后的自动代理生成器标志还是牵线人进行切面类和目标类.业务函数的联系。

  6. execution简化

    1. 去掉public
    2. 用*..
    3. ..代替arg
    4. do*代替业务函数

一、防火墙

  1. 查看状态 systemctl status firewalld.service
  2. 关闭systemctl stop/disable firewalld.service
  3. 重启防火墙 systemctl restart firewalld

二、SSH

  1. 查看状态 systemctl status sshd.service
  2. 同防火墙

三、网络

  1. 关闭

    ​ 在CentOS系统上，目前有NetworkManager和network两种网络管理工具。如果两种都配置会引起冲突，而且NetworkManager在网络断开的时候，会清理路由，如果一些自定义的路由，没有加入到NetworkManager的配置文件中，路由就被清理掉，网络连接后需要自定义添加上去。
    ​ 目前在CentOS上的NetworkManager版本比较低，而且比较适合有桌面环境的系统，所以服务器上保留network服务即可，将NetworkManager关闭，并且禁止开机启动。

  2. 开启网络 systemctl start netword.service

四、端口

  1. 查看端口是否开放

    firewall-cmd –query-port=1521/tcp

  2. 查看所有开放的端口

    firewall-cmd –zone=public –list-ports

  3. 开启、关闭某个端口

    firewall-cmd –zone=public –add-port=1521/tcp –permanent

    firewall-cmd –zone=public –remove-port=1521/tcp –permanent

  4. 开启、关闭连续端口

    firewall-cmd –zone=public –add-port=1000-2000/tcp –permanent

    firewall-cmd –zone= public –remove-port=1000-2000/tcp –permanent

  5. 重新加载配置文件

    firewall-cmd –reload

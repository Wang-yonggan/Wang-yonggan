equals和==

  1. ==
    • 对于基本类型，==号比较的是是值相等
    • 对于引用类型比较的就是是否为同一个对象: id
  2. equals: 自己定义的等比较,一般先确定是否为同一个对象，就是＝＝，然后判断是否为同一个类，然后判断细节
    1. String
      1. 先判断是否为同类
      2. 每个index的值都相等
    2. Integer
      1. 先判断是否为同类
      2. 比较value

hashcode

  1. String：每个Σ value*2^index
  2. Integer long byte： 都是返回值本身
  3. Object：native方法，根据地址计算得到的

获得地址

Integer

  1. Integer a=1,b=1,new c=1则永远有 a==b,a.equals(b),因为==号基本类型比较的是值
  2. 但是他们的地址不一样
  3. new创建的方式地址永远不一样
  4. 常量池问题
    1. Integer a=1; 其实是Integer.valueOf(1)的简化。
    2. Integer.valueOf(1)会从Integer(-128,127)中选出1
    3. 这样他们其实是同一个对象，地址也一样

  2. 线程和进程概念
    1. 进程与线程
      1. 进程是分配资源的基本单元，运行的一个程序
      2. 线程是cpu调度的基本单元，程序执行的基本单位
    2. 线程的状态
      1. 新建
      2. 就绪
      3. 等待
      4. 运行
      5. 死亡
    3. wati和sleep
      1. Object.wait
        1. 释放锁
      2. Thread.sleep
        1. 不释放当前锁
    4. 并发和并行
      1. 串行：排队执行
      2. 并行：一并执行
      3. 并发：一个cpu表面上运行多个线程
      4. 并行：多个cpu
  3. Lock
    1. synchronize，同步锁
      1. 修饰代码块
        1. 作用于对象是调用这个代码块的对象
      2. 修饰方法
        1. 被修饰的方法称为同步方法，其
        2. 作用的对象是调用这个方法的对象；
      3. 修饰静态方法
        1. 作用的对象是这个类的所有对象
      4. 修饰类
        1. 作用主的对象是这个类的所有对象
    2. lock和synchronized区别
      1. lock可获得更广泛的锁
      2. lock是类，不是java内置
      3. lock是手动锁，synchronized是自动锁
      4. synchronized发生异常或者结束会自动释放，而lock发生了异常会一直占用锁
      5. lock可以判断是否得到了锁
      6. lock可以在竞争激烈的情况下提高效率
  4. 管程:
    1. Monitor、监视器、锁
    2. 同步机制，保证只有一个线程能访问资源
  5. 用户线程和守护线程
    1. 用户线程：普通的线程，main结束了，用户线程还会继续运行
    2. 守护线程：后台默默运行的线程 isDaemon，main结束，收回线程会结束
  6. Java Thread的start和run方法的区别
    1. start()会导致run()方法被调用
    2. run()方法中的内容称为线程体，它就是这个线程需要执行的工作。
      用start()来启动线程，实现了真正意义上的启动线程，此时会出现异步执行的效果，即在线程的创建和启动中所述的随机性。 而如果使用run()来启动线程，就不是异步执行了，而是同步执行，不会达到使用线程的意义

注解

例子
@Conditional{a.class} Spring4,需要一组Condition.class参数，只有a.class的matches返回true才会创建bean到ioc
@ConditionalOnClass(a.class) 只有a.class才创建，这不是多此一举，而是因为该注解支持用字符串
@ConditionalOnBean(“a”) 当bean a存在时，才创建一个默认的Bean，一般用于创建依赖beana的bean
@ConditionalOnMissingBean(“a”) 当bean a不存在时，才创建bean，一般用于创建同类bean
@ConditionalOnExpression 根据spring EL表达式判断 @ConditionalOnProperty(name = “ep.cache”, havingValue = “true”)
@ConditionalOnProperty 基于application.properties的k-v判断
@DependsOn(“a”) name为a的bean加载完后再加载该bean，用于定义bean加载顺序
@AutoConfigureBefore 这三个注解需要配合spring.factory才能指定顺序，并且不能被扫描到
@AutoConfigureAfter{a.class} configruation a被加载之后在加载该configruation
@AutoConfigureOrder 定义configruation的加载顺序

一、磁盘

1.1 磁盘组成

  1. 扇区–>磁道–>片面–>盘面–>磁盘

  2. 磁头–>磁臂–>转轴

1.2 读取时间

读取时间= 寻道时间+旋转延迟+数据传输时间

  1. 寻道时间：磁头到数据所在磁道所需时间，大约10ms左右。
  2. 旋转延迟：扇区旋转到磁头下方所需时间，决于磁盘转速，转速一般6000rpm左右，大约60*1000/6000=10ms。
  3. 数据传输时间：完成传输所请求的数据所需要的时间。

1.3 扇区、块/簇、页page

扇区 –>块/簇–>page

  1. 扇区： 硬盘的最小读写单元，512字节
  2. 块/簇： 操作系统针对硬盘读写的最小单元，是逻辑概念，一块包含2*n个扇区
  3. page： 是内存与操作系统之间交互的最小单元。

1.4 磁盘预读

 当一个数据被用到时，其附近的数据也通常会马上被使用。由于磁盘顺序读取的效率很高（不需要寻道时间，只需很少的旋转时间），因此对于具有局部性的程序来说，预读可以提高I/O效率。预读的长度一般为**页（page）**的整倍数。页是计算机管理存储器的逻辑块，大小通常为4k，主存和磁盘以页为单位交换数据。当程序要读取的数据不在主存中时，会触发一个缺页异常，此时系统会向磁盘发出读盘信号，磁盘会找到数据的起始位置并向后连续读取一页或几页载入内存中，然后异常返回，程序继续运行。

二、page cache

页缓存

页缓存是操作系统为了提升磁盘读写性能在应用进程与磁盘之间加设的提供预读和异步刷盘机制的内核缓冲区。java 的 IO 操作是建立在操作系统的 IO 之上的，从最基础的 read/write 系统调用，到具有零拷贝特性的 sendfile、mmap，在 java 中都能看到它们的身影。本文的主要目的是纵观全局，鸟瞰 java IO 体系，并指出每种 IO 方式的特点与使用场景。整篇文章会围绕下图作分步讲解，为了简单起见，这里主要以写操作为例。

注解 作用
@EnableScheduling 开启定时任务功能 configuration
@Scheduled(cron=”0/10 * * * * ? “) 指定cron表达式 method
@Schedules 指定多个周期 method
@EnableAsync 开启异步支持 Configuration
@Async 异步任务，spring自带的线程池(可以修改)调用该方法 Method

@Scheduled

  1. @Scheduled(fixedRate=2000) // 每隔执行时间+2秒执行一次
  2. @Scheduled(fixDelay=2000) //每隔2秒一次
  3. @Scheduled(cron=”0/10 * * * * ? “) // 每隔10秒执行一次
  4. @Scheduled(cron=”0 33/3 * * * ?”) //每小时的33分钟开始执行，每3分钟执行1次

异步、@Async失效

  1. 异步方法使用static修饰
  2. 异步类没有使用@Component注解（或其他注解）导致spring无法扫描到异步类
  3. 异步方法不能与异步方法在同一个类中
  4. 类中需要使用@Autowired或@Resource等注解自动注入，不能自己手动new对象
  5. 如果使用SpringBoot框架必须在启动类中增加@EnableAsync注解
  6. 在Async 方法上标注@Transactional是没用的。 在Async 方法调用的方法上标注@Transactional 有效。
  7. 调用被@Async标记的方法的调用者不能和被调用的方法在同一类中不然不会起作用！！！！！！！
  8. 使用@Async时要求是不能有返回值的不然会报错的 因为异步要求是不关心结果的

优点

  • 使用简单。
  • 支持cron表达式，支持固定频率执行。
  • 代码侵入性低。

缺点

  • 无法解决分布式调度问题。
  • 无法监控任务状态。
  • 无其他骚功能。如失败提醒，重试等等。

内部类

发表于 2022-03-22 Changyan：
本文字数： 2.4k 阅读时长 ≈ 2 分钟

1 作用

  1. 可以实现多重继承：定义一个半成品类继承类1，
  2. 可以直接访问外部类的成员，包括私有

2 内部类的分类

  • 成员内部类 –> 类里的类

    不能定义静态变量，除了final修饰的

  • 局部内部类 –> 函数里面的类

  • 静态内部类 –> 类里面static类

    和普通的类差不多,引用方式 Out.Inner inner = new Out.Inner(); inner.do();

  • 匿名内部类 –> lambda

3 注意点

  1. 各个类也有访问权限

    1. 默认defalut
    2. 只有public的能在别的包创建这个内部类

  2. 非static的成员内部类不能有static方法,也即不能有main函数

  3. 创建一个成员内类就相当于创建了两个类: 外部+内部

  4. 函数里面不能有权限符号 private等,所有局部内部类都是只有这个方法能访问的

  5. 为什么局部内部类访问局部变量必须加final修饰呢

    ​ 因为局部变量是随着方法的调用而调用，使用完毕就消失，而堆内存的数据并不会立即消失。

    所以，堆内存还是用该变量，而该变量已经没有了。为了让该值还存在，就加final修饰。

    原因是，当我们使用final修饰变量后，堆内存直接存储的是值，而不是变量名。

2. 成员内部类

成员内部类不能定义静态变量。类初始化的时候先初始化静态成员，如果允许成员内。部类定义静态变量，那么成员内部类的静态变量初始化顺序是有歧义的

HDFS集群中，有两种节点，分别为Namenode，Datanode；

  • Namenode它的作用时记录元数据信息，记录块信息和对节点进行统一管理。比如用户要存储一个很大的文件，HDFS系统会对这个文件进行切分，然后存储在多台Namenode节点当中，那么每个切的大小，存储的路径信息，文件的副本数等元数据信息会存储在元数据当中，由Namenode进行管理和记录。
  • Datanode节点的作用是存储数据，Namenode将数据切块后的分配给多个Datanode节点，Datanode对数据块进行存储，

线程池的作用

1.减少了创建和销毁线程的次数，每个工作线程都可以被重复利用，可执行多个任务。

2.可以根据系统的承受能力，调整线程池中工作线线程的数目，防止因为消耗过多的内存，而把服务器累趴下(每个线程需要大约1MB内存，线程开的越多，消耗的内存也就越大，最后死机)。

参数 ThreadPoolExecutor

参数
int corePoolSize 常驻线程数量
int maximumPoolSize 表示最大线程数，如当前任务较多，将核心线程数都用完了，此时就会创建新的线程，但线程池内线程总数不会超过最大线程数
long keepAliveTime 空闲线程存活时间(减少到常驻数量)
TimeUnit unit 时间类型
BlockingQueue workQueue 多的线程进入–>阻塞队列
ThreadFactory threadFactory 线程工厂
RejectedExecutionHandler handler 当队列满的时候，拒绝策略

二、类型

2.1 定长线程池

多出的线程进入LinkedBlockingQueue，

2.2 一池一线程

2.3 可扩容池

  • 它比较适合处理执行时间比较小的任务；

2.4 定长延迟线程池

可周期性，延迟性

三、queue类型

SynchronousQueue

没有容量，直接提交。

SynchronousQueue阻塞队列一般要求maximumPoolSizes为无界，避免线程拒绝执行操作。

LinkedBlockingQueue

无界队列，多于corePoolSize的线程被放进来等待，maximumPoolSizess变为无效参数

ArrayBlockingQueue

有界队列，可以指定缓存队列的大小，当线程数量大于corePoolSize时，多余的任务会缓存在ArrayBlockingQueue队列中等待有空闲的线程时继续执行；当ArrayBlockingQueue满时，则又会开启新的线程去执行，直到线程数量达到maximumPoolSize；当线程数已经达到最大的maximumPoolSize时，再有新的任务到达时会执行拒绝执行策略

四、其他

4.2 拒绝策略

名称 作用
CallerRunsPolicy 使用调用线程直接运行任务
AbortPolicy 丢弃
DiscardPolicy 丢弃，抛出异常
DiscardOldestPolicy 丢弃阻塞队列 workQueue 中最老的一个任务，并将新任务加入

4.2 ThreadFactory

线程池可选参数，在线程放入线程池前处理线程

  1. 设置名称
  2. 设置是否为守护线程
  3. 设置优先级

synchonized和Lock区别

  1. synchonized是一个关键字，Lock是一个类
  2. synchonized是隐式锁，自动上锁解锁，Lock是显式锁，手动操作
  3. synchonized无法获得所得状态，Lock可以判断是否获得了锁，更灵活
  4. synchonized是非公平锁，Lock默认是非公平锁，可设置
  5. synchonized适合锁少量的代码，Lock是否大量代码
  6. conditon可以精准唤醒，可以灵活调整获得锁的顺序

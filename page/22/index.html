内部类

发表于 2022-04-27 更新于 2022-03-22 Changyan：
本文字数： 2.4k 阅读时长 ≈ 2 分钟

1 作用

  1. 可以实现多重继承：定义一个半成品类继承类1，
  2. 可以直接访问外部类的成员，包括私有

2 内部类的分类

  • 成员内部类 –> 类里的类

    不能定义静态变量，除了final修饰的

  • 局部内部类 –> 函数里面的类

  • 静态内部类 –> 类里面static类

    和普通的类差不多,引用方式 Out.Inner inner = new Out.Inner(); inner.do();

  • 匿名内部类 –> lambda

3 注意点

  1. 各个类也有访问权限

    1. 默认defalut
    2. 只有public的能在别的包创建这个内部类

  2. 非static的成员内部类不能有static方法,也即不能有main函数

  3. 创建一个成员内类就相当于创建了两个类: 外部+内部

  4. 函数里面不能有权限符号 private等,所有局部内部类都是只有这个方法能访问的

  5. 为什么局部内部类访问局部变量必须加final修饰呢

    ​ 因为局部变量是随着方法的调用而调用，使用完毕就消失，而堆内存的数据并不会立即消失。

    所以，堆内存还是用该变量，而该变量已经没有了。为了让该值还存在，就加final修饰。

    原因是，当我们使用final修饰变量后，堆内存直接存储的是值，而不是变量名。

2. 成员内部类

成员内部类不能定义静态变量。类初始化的时候先初始化静态成员，如果允许成员内。部类定义静态变量，那么成员内部类的静态变量初始化顺序是有歧义的

一、IO概述

  1. 同步阻塞IO-面向流-BIO
    1. 一个请求连接一个线程，
    2. 问题：需要创建很多的线程，所有的线程还是同步的
    3. 解决：线程池，但是线程池数量有上限
    4. 适合请求少的场景，程序简单
  2. 同步非阻塞IO-多路复用器-NIO
    1. 一个线程可以处理多个请求
    2. selector 选择器(复用器)
    3. 适合请求多，请求时间短：聊天，弹幕
  3. 异步非阻塞IO–AIO(NIO2.0)
    1. 一个请求一个线程
    2. 适合请求较多，且请求时间长场景

2.2.3 缺陷

  • 适应：小批量，请求处理快速的场景

3.3 Channel

Channel代表这一个网络连接通道，当Channel注册到Selector后会返回一个SelectionKey对象，该SelectionKey对象则代表这这个Channel和它注册的Selector间的关系。并且SelectionKey中维护着两个很重要的属性：interestOps、readyOps。

interestOps是我们希望Selector监听Channel的哪些事件。我们将我们感兴趣的事件设置到该字段，这样在selection操作时，当发现该Channel有我们所感兴趣的事件发生时，就会将我们感兴趣的事件再设置到readyOps中，这样我们就能得知是哪些事件发生了以做相应处理。

3.3.1 特点

  1. 双向的
  2. 不能直接数据交互，需借助buffer
  3. 异步读写

3.3.2 Channel实现

  1. FileChannel 文件
  2. DatagramChannel UDP
  3. SocketChannel TCP
  4. ServerSockeChannel 监听Web的TCP连接，每一个连接创建一个Channel

3.4 Selector

选择器，Selector是NIO中实现I/O多路复用的关键类。Selector实现了通过一个线程管理多个Channel，从而管理多个网络连接的目的，

3.4.1 工作方式

多个 Channel 以事件的方式可以注册到同一个Selector，如果有事件发生，便获取事件然后针对每个事件进行相应的处理。这样就可以只用一个单线程去管理多个通道，也就是管理多个连接和请求。避免了多线程之间的上下文切换导致的开销

3.4.2 选择器作用

  1. 轮询Channel选出可用的Channel

一、Git概念

  1. 手工式版本控制

    1. 特点
      1. 每改一次备份一次
      2. 协同更改需要同时U盘copy
    2. 问题

      1. 备份

      2. 代码还原

      3. 协同开发

      4. 代码追溯(追责)

  2. 解决：版本控制结束了手动管理多个“版本”的史前时代，进入到版本控制的20世纪。

  3. 集中式版本控制SVN/CVS

    1. 版本库都放在集中式服务器
    2. 使用前先从集中式服务器copy过来
    3. 修改完后在上传到集中服务器

  4. 分布式版本控制（共享版本库）

    1. 每个人都有所有的版本，gitee/github也算个人
    2. 一个人修改了A那么直接上传就会记录这个修改内容
    3. 本地有所有的版本

三、远程仓库

拉取远程仓库的更新

  • 执行 git fetch origin master 时，它的意思是从名为 origin 的远程上拉取名为 master 的分支到本地分支 origin/master 中
  • 执行 git merge origin/master 时，它的意思是合并名为 origin/master 的分支到当前所在分支。
  • 执行 git push origin master 时，它的意思是推送本地的 master 分支远程 origin，
  • 还可以一次性拉取多个分支的代码：git fetch origin master stable oldstable；
  • 也还可以一次性合并多个分支的代码：git merge origin/master hotfix-2275 hotfix-2276 hotfix-2290；

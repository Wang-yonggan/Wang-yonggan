一、IO概述

  1. 同步阻塞IO-面向流-BIO
    1. 一个请求连接一个线程，
    2. 问题：需要创建很多的线程，所有的线程还是同步的
    3. 解决：线程池，但是线程池数量有上限
    4. 适合请求少的场景，程序简单
  2. 同步非阻塞IO-多路复用器-NIO
    1. 一个线程可以处理多个请求
    2. selector 选择器(复用器)
    3. 适合请求多，请求时间短：聊天，弹幕
  3. 异步非阻塞IO–AIO(NIO2.0)
    1. 一个请求一个线程
    2. 适合请求较多，且请求时间长场景

每个请求创建一个线程去处理请求

2.2.3 缺陷

  1. 每个请求都创建线程，线程的切换、竞争会浪费资源
  2. 线程本身也占用cpu资源
  3. 并不是每个请求线程

2.3 伪异步

  • 优：创建线程池，能避免创建线程的资源消耗问题
  • 缺：底层还是阻塞的，如果请求处理较慢，那么新的请求就会长时间的再请求队列中，产生大量超时
  • 适应：小批量，请求处理快速的场景

3.3 Channel

Channel代表这一个网络连接通道，当Channel注册到Selector后会返回一个SelectionKey对象，该SelectionKey对象则代表这这个Channel和它注册的Selector间的关系。并且SelectionKey中维护着两个很重要的属性：interestOps、readyOps。

interestOps是我们希望Selector监听Channel的哪些事件。我们将我们感兴趣的事件设置到该字段，这样在selection操作时，当发现该Channel有我们所感兴趣的事件发生时，就会将我们感兴趣的事件再设置到readyOps中，这样我们就能得知是哪些事件发生了以做相应处理。

3.3.1 特点

  1. 双向的
  2. 不能直接数据交互，需借助buffer
  3. 异步读写

3.3.2 Channel实现

  1. FileChannel 文件
  2. DatagramChannel UDP
  3. SocketChannel TCP
  4. ServerSockeChannel 监听Web的TCP连接，每一个连接创建一个Channel

3.4 Selector

选择器，Selector是NIO中实现I/O多路复用的关键类。Selector实现了通过一个线程管理多个Channel，从而管理多个网络连接的目的，

3.4.1 工作方式

多个 Channel 以事件的方式可以注册到同一个Selector，如果有事件发生，便获取事件然后针对每个事件进行相应的处理。这样就可以只用一个单线程去管理多个通道，也就是管理多个连接和请求。避免了多线程之间的上下文切换导致的开销

java四大域对象-转载

  1. request HttpServletContext

    1. 生命周期：

      在Service方法调用前由服务器创建，传入service方法。整个请求结束，request生命结束。
      用户发送一个请求，开始，服务器返回响应，请求结束，生命周期结束；

    2. 作用范围：整个请求链（请求转发也存在）/一次请求转发的范围

    3. 作用：获取请求数据，共享数据，请求转发。

  2. session HttpSession

    1. 生命周期

      在第一次调用request.getSession()方法时，服务器会检查是否已经有对应的session，如果没有就在内存中创建一个session并返回。
      （1）当一段时间内session没有被使用（默认为30分钟），则服务器会销毁该session。
      （2）如果服务器非正常关闭，没有到期的session也会跟着销毁。
      （3）如果调用session提供的invalidate()，可以立即销毁session。

  3. application ServletContext

    1. 生命周期：整个web服务
    2. 作用范围：整个web应用
    3. 作用：
      1. 读取配置文件
      2. 在不同Servlet 之间转发

  4. page(jsp) pageContext

    1. 可以作为入口对象获取其他三大域对象的引用，获取其所有域中的数据
    2. 作用于整个JSP页面，是四大作用域中最小的一个

  5. 特点

    1. 之所以他们是域对象，原因是他们都内置了map集合，都有setAttribute和getAttribute方法。
    2. 都有生命周期，servlet{init,service,destory}也有生命周期。

  6. 如果一个数据只在当前jsp页面使用，可以使用pageContext域

    如果一个数据，除了在当前Servlet中使用，还要在请求转发中使用，使用request域

    如果一个数据，除了现在我自己要用，过一会我自己还要用，存在session域

    如果一个数据，除了我自己用其他人也要用，存在ServleContext域中

  7. 域对象：能够在自己的作用范围共享数据。

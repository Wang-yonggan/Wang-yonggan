2.3 explain参数
id 指定父子查询，id大的先执行，如果没有子查询，一般都是1
select_type 查询类型(父查询，子查询，普通查询，联合查询)
table 使用的表别名
type all: 全表扫描 index:遍历所引述 range:
ref做分界线 system > const > eq_ref > [ref] > fulltext > ref_or_null > index_merge > unique_subquery > index_subquery > range > index > ALL
possiable_keys 可能使用的索引
key 实际用的索引
key_len 当前使用索引的长度
ref
rows 需要扫描的行数
filtered 表示符合条件条数占比
extra useTemp 使用临时表，useFileSory 使用文件排序(需要被优化)
名称 描述 例子
查询时间 < 扫描时间
system 查询系统表 select * from mysql.time_zone
const where主键索引或唯一索引，且为常量 select * from user where id=1
eq_ref 用到了唯一索引 数量很多
ref 用到了不唯一索引，join 索引=外键索引 select * from u1,u2 where u1.name=u2.name
range 索引范围查询，避免了扫描索引 select * from user where id < 10
index 扫描全部索引文件，且不需要回表(索引覆盖) select count(*) from user
all 没命中索引全表扫描 select * from user where age=10
select * from user limit 2

五、mysql存储过程

存储例程是存储在数据库服务器中的一组sql语句，通过在查询中调用一个指定的名称来执行这些sql语句命令

存储过程的优点

  1. 减少网络通信量
  2. 执行速度更快
    1. 在存储过程创建的时候，数据库已经对其进行了一次解析和优化
    2. 存储过程会(备份)保存在内存中
  3. 更强的适应性

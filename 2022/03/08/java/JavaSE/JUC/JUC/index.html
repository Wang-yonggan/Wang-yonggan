一、线程和进程概念

1.1 进程与线程

  1. 进程是分配资源的基本单元，运行的一个程序
  2. 线程是cpu调度的基本单元，程序执行的基本单位
  3. 管程:
    1. Monitor、监视器、锁
    2. 同步机制，保证只有一个线程能访问资源
  4. 用户线程和守护线程
    1. 用户线程：普通的线程，main结束了，用户线程还会继续运行
    2. 守护线程：后台默默运行的线程 isDaemon，main结束，收回线程会结束

1.2 线程的状态

  1. 新建
  2. 就绪
  3. 等待
  4. 运行
  5. 死亡

1.3 wati和sleep

  1. Object.wait
    1. 释放锁
  2. Thread.sleep
    1. 不释放当前锁

1.4 并发和并行

  1. 串行：排队执行
  2. 并行：一并执行
  3. 并发：一个cpu表面上运行多个线程
  4. 并行：多个cpu

1.5 Lock、synchronize

1.5.1 synchronize，同步锁

  1. 修饰代码块
    1. 作用于对象是调用这个代码块的对象
  2. 修饰方法
    1. 被修饰的方法称为同步方法，其
    2. 作用的对象是调用这个方法的对象；
  3. 修饰静态方法
    1. 作用的对象是这个类的所有对象
  4. 修饰类
    1. 作用主的对象是这个类的所有对象

1.5.2 lock和synchronized区别

  1. lock可获得更广泛的锁

  2. lock是类，不是java内置

  3. lock是手动锁，synchronized是自动锁

  4. synchronized发生异常或者结束会自动释放，而lock发生了异常会一直占用锁

  5. lock可以判断是否得到了锁

  6. lock可以在竞争激烈的情况下提高效率

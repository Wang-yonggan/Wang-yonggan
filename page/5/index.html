一、索引原理

索引原理：

索引是一种利用某种规则的数据结构与实际数据的关系加快数据查找的功能；索引数据节点中有着实际文件的位置，因为索引是根据特定的规则和算法构建的,在查找的时候遵循索引的规则可以快速查找到对应数据的节点，从而达到快速查找数据的效果；其实宏观来说索引其实是一种概念而不是具体的某项技术，只是我们在某个技术中运用得比较广泛和鲜明（比如说数据库）渐渐的有了特定领域的标签，其实在生活中索引的使用无处不在，比如说：书本里的目录；读书时的座位号，考试编号都有类似索引的功能;

总结来所有通过某规则数据结构和实际目标关联，根据特定规则算法快速寻址的功能都可以称之为索引；

正好命中索引，不需要回表

select id,name from user 索引覆盖的前提是name有索引，因为name的索引表中存储的是（name,id），一个表就存储了name和id

二、索引结构

1.1 为什么是b+树

  1. 索引的表象是一个目录
  2. 底层原理 [字段值,地址] 某种数据结构的映射

1.2 排序的结构

  1. 二叉树 —> 递增有序的情况 树高度==n
  2. AVL 平衡二叉树 旋转次数多
  3. 红黑树 弱平衡二叉树 弊端: 高度高
  4. B树 m叉树+平衡查找树 扩大横分叉，高度可控
  5. B+树 每个节点从左到右一次递增
    1. 非叶子节点不存数据，只存冗余索引
    2. 叶子节点从大到小相互连接
    3. 每个页16KB
      1. 如果满了 节点分裂 高度增加

B+树相对于B树优点

  1. 非叶子节点都存有冗余，这样的目的是为了让叶子节点存有所有的数据
  2. 非叶子节点不带有指向记录的指针，则一个块可以存储更多的索引项，如此可以降低树的高度。
  3. 叶子节点之间有指针连接，则在范围扫描时避免了在内部节点来回移动。

B+树搜索过程

  1. 根据冗余索引找到data叶子节点
  2. 把data加载到内存 然后二分查找

特别的

  1. 如果只有一层 那么就相当于二分查找 ，但是，但是这意味着把所有的数据加载到内存，内存吃不消。
  2. hash表
    1. 数组+链表
    2. hash运算—>存到链表(hash冲突直接加在链表尾部)
    3. 不支持范围查找
    4. 不支持模糊查询
    5. 大量重复数据，会严重影响查询性能
  3. mysql的memory支持hash索引，一般hash索引用于名字，而不用于数字
  4. innodb不支持hash索引，

四、索引创建原则

需要建立索引

  1. 该字段经常select+where, update+where delete where
  2. 需要分组、排序的字段
  3. 唯一的字段

不需要建立索引

  1. 数据量小不建立

  2. 经常update,delete,insert

  3. 不经常where

  4. 数据重复严重

    索引的目的是为了加速，比如把sex建立索引，加速太小，反而还需要维护索引，不如全表扫描

  5. 字段长度太大

一、网络编程概念

socket就是对UDP、TCP的封装

1.1 TCP UDP

  • TCP：传输控制协议，是一种面向连接的、可靠的、基于字节流的传输层通信协议，TCP 层是位于 IP 层之上，应用层之下的中间层。TCP 保障了两个应用程序之间的可靠通信。通常用于互联网协议，被称 TCP / IP。
  • UDP：用户数据报协议，位于 OSI 模型的传输层。一个无连接的协议。提供了应用程序之间要发送数据的数据报。由于UDP缺乏可靠性且属于无连接协议，所以应用程序通常必须容许一些丢失、错误或重复的数据包。不局限于端到端，可以广播

1.2 socket概念

  1. 在计算机网络中，叫套接字socket (ip:port)

    top连接={socket1,socket2}

  2. 在程序通信中，叫socket Api，网络通信接口，这是对TCP、IP的封装

1.3 socket编程

当连接建立时，服务器会创建一个 Socket 对象。客户端和服务器现在可以通过对 Socket 对象的写入和读取来进行通信。

1.4 IO模型

  1. 阻塞IO

    进程进行IO操作的适合被挂起，防止被运行，指导IO操作完成。

  2. 非阻塞IO

    IO操作立即返回，返回值为读取到的数据大小。等到总线发送了中断信号，就表示IO操作完成。

1.5 Java中的net

​ 网络编程的基本模型是Client/Server模型，也就是两个进程之间进行相互通信，客户端 - 服务端是完全同步，完全耦合的

  • 服务端ServerSocket，负责绑定IP地址和端口，提供位置信息
  • 客户端Socket，通过连接操作，向服务端监听的端口地址发起连接请求
  • 基于TCP协议下进行三次握手连接
  • 连接成功后，双方通过网络套接字（Socket）（输入和输出流）进行同步阻塞式通信

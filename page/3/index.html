二级缓存失效

  1. sqlSession没有close
  2. cacheKey不同
  3. select –> update –> select

SerializedCache

如果readOnly=false,可读写的缓存 会返回缓存对象的拷贝(通过序列化) 。这会慢一些,但是安全,因此默认是 false。

SynchronizedCache

同步缓存, 3.2.6以后这个类已经没用了，考虑到Hazelcast, EhCache已经有锁机制了，所以这个锁就画蛇添足了。

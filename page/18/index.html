一、网络编程概念

socket就是对UDP、TCP的封装

1.1 TCP UDP

  • TCP：传输控制协议，是一种面向连接的、可靠的、基于字节流的传输层通信协议，TCP 层是位于 IP 层之上，应用层之下的中间层。TCP 保障了两个应用程序之间的可靠通信。通常用于互联网协议，被称 TCP / IP。
  • UDP：用户数据报协议，位于 OSI 模型的传输层。一个无连接的协议。提供了应用程序之间要发送数据的数据报。由于UDP缺乏可靠性且属于无连接协议，所以应用程序通常必须容许一些丢失、错误或重复的数据包。不局限于端到端，可以广播

1.2 socket概念

  1. 在计算机网络中，叫套接字socket (ip:port)

    top连接={socket1,socket2}

  2. 在程序通信中，叫socket Api，网络通信接口，这是对TCP、IP的封装

1.3 socket编程

当连接建立时，服务器会创建一个 Socket 对象。客户端和服务器现在可以通过对 Socket 对象的写入和读取来进行通信。

1.4 IO模型

  1. 阻塞IO

    进程进行IO操作的适合被挂起，防止被运行，指导IO操作完成。

  2. 非阻塞IO

    IO操作立即返回，返回值为读取到的数据大小。等到总线发送了中断信号，就表示IO操作完成。

1.5 Java中的net

​ 网络编程的基本模型是Client/Server模型，也就是两个进程之间进行相互通信，客户端 - 服务端是完全同步，完全耦合的

  • 服务端ServerSocket，负责绑定IP地址和端口，提供位置信息
  • 客户端Socket，通过连接操作，向服务端监听的端口地址发起连接请求
  • 基于TCP协议下进行三次握手连接
  • 连接成功后，双方通过网络套接字（Socket）（输入和输出流）进行同步阻塞式通信

1.6.1 flush

  2. 但是有时候不足1KB需要使用flush才发送

三、步骤

  1. 创建一个ServerSocket，监听并绑定一个端口
  2. 一系列客户端来请求这个端口
  3. 服务器使用Accept，获得一个来自客户端的Socket连接对象
  4. 启动一个新线程处理连接
    1. 读Socket，得到字节流
    2. 解码协议，得到Http请求对象
    3. 处理Http请求，得到一个结果，封装成一个HttpResponse对象
    4. 编码协议，将结果序列化字节流 写Socket，将字节流发给客户端
  5. 继续循环步骤3
